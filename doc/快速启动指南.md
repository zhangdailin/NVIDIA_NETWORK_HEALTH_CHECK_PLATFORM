# 快速启动指南

## 启动步骤

### 1. 启动后端服务

```bash
cd backend
python -m uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

后端服务将在 `http://localhost:8000` 启动

### 2. 启动前端服务

**开发模式:**
```bash
cd frontend
npm run dev
```

**生产模式:**
```bash
cd frontend
npm run build
npm run preview
```

前端服务将在 `http://localhost:5173` (dev) 或 `http://localhost:4173` (preview) 启动

### 3. 访问应用

打开浏览器访问: `http://localhost:5173`

## 使用流程

### 第一步: 上传IBDiagnet数据
1. 在左侧边栏点击 **IBDiagnet Analysis** 上传区域
2. 选择 `.zip` 或 `.tar.gz` 格式的ibdiagnet输出文件
3. 等待上传和分析完成 (通常需要10-30秒)

### 第二步: 查看故障汇总
分析完成后,自动显示 **Overview** 页面,顶部即为 **故障汇总**

#### 如果网络健康
显示:
```
✅ 网络健康状况良好
未发现严重问题或警告
```

#### 如果有故障
显示分类汇总:
- 🔴 **严重故障** - 需要立即处理
- ⚠️ **警告** - 需要关注
- ℹ️ **提示** - 配置建议

### 第三步: 查看详细信息
1. 点击任意故障类别展开列表
2. 点击故障项的 **[查看详情]** 按钮
3. 阅读知识库提供的:
   - 问题描述
   - 可能原因
   - 修复步骤
   - 预计修复时间

### 第四步: 深入分析 (可选)
点击顶部Tab切换到具体分析页面:
- **Cable Issues**: 线缆和光模块详细信息
- **Congestion**: 拥塞详细分析
- **BER**: 误码率详细分析
- **Firmware**: 固件版本分析
- 等等...

## 示例工作流

### 日常巡检
1. 上传最新ibdiagnet数据
2. 查看Overview故障汇总
3. 记录严重故障数量
4. 生成巡检报告

### 故障排查
1. 上传问题时段的ibdiagnet数据
2. 在故障汇总中定位严重问题
3. 展开详情获取修复建议
4. 切换到详细Tab查看原始数据
5. 按建议步骤修复

### 性能优化
1. 上传基线数据
2. 查看警告级别的性能问题
3. 分析拥塞、延迟等指标
4. 制定优化计划

## 常见问题

### Q: 上传失败怎么办?
**A:** 检查:
1. 文件格式是否为 .zip 或 .tar.gz
2. 文件大小是否超过500MB
3. 压缩包内是否包含 `.db_csv` 文件
4. 后端服务是否正常运行

### Q: 分析很慢怎么办?
**A:** 正常情况下:
- 小型网络 (< 100节点): 5-10秒
- 中型网络 (100-500节点): 10-30秒
- 大型网络 (> 500节点): 30-60秒

如果超过这个时间,检查:
1. 后端日志是否有错误
2. 服务器资源是否充足

### Q: 没有显示故障汇总?
**A:** 可能原因:
1. 网络确实健康,会显示"✅ 网络健康状况良好"
2. 检查浏览器控制台是否有错误
3. 刷新页面重试

### Q: 如何导出分析结果?
**A:**
1. 使用浏览器的打印功能 (Ctrl+P)
2. 选择"保存为PDF"
3. 或者截图保存故障汇总部分

## 系统要求

### 后端
- Python 3.8+
- 4GB+ RAM
- 2+ CPU cores

### 前端
- Node.js 16+
- 现代浏览器 (Chrome/Edge/Firefox)

## 性能建议

### 优化上传速度
- 使用有线网络
- 确保后端在本地网络
- 压缩包尽量小于100MB

### 优化分析速度
- 为后端分配更多CPU
- 使用SSD存储
- 定期清理uploads目录
